Generics

Background

Generics remove the need for casting. As we have seen, ArrayLists, are great with managing a large set of objects.  However, everything is stored as an "Object", there is a performance penalty up and down casting every time you add and remove an object from the collection. Generics are one the many great features added in .NET 2.0 that made C# a very powerful language. Generics allow type checking and the related benefits to integrate into the collection classes.  We'll see some more advanced C# features in future lectures.

  • Use generic types to maximize code reuse, type safety, and performance.
  • The most common use of generics is to create collection classes.
  • The .NET Framework class library contains several new generic collection classes in the System.Collections.Generic namespace. These should be used whenever possible instead of classes such as ArrayList in the System.Collections namespace.
  • You can create your own generic interfaces, classes, methods, events and delegates.
  • Generic classes may be constrained to enable access to methods on particular data types
  • Information on the types that are used in a generic data type may be obtained at run-time by using reflection.
