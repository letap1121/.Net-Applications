Queues, Stacks and Dictionaries

Background

There are speciality Collections which are useful to explore and are frequently used.

Queues

A first-in, first-out collection of objects.

 Queue<string> queue = new Queue<string>();
//Add Item to Queue
queue.Enqueue("A");
//Remove item from Queue
Console.WriteLine(queue.Dequeue());

See Example: QueueExample

Stacks

Represents a simple last-in-first-out (LIFO) collection of objects.

//Initialize Stack
Stack<string> stack = new Stack<string>();
//Add Item to Stack
stack.Push("A");
//Remove item from Stack
Console.WriteLine(stack.Pop());

See Example: StackExample

To see a visual example of the difference between stack and queue see Example: StackVsQueueWPF.

Dictionaries

Represents a collection of keys (unique) and values. Main advantage is that a Dictionary provides for quick lookups based on key.

Code fragements:

Dictionary<String, int> wordDictionary = new Dictionary<string, int>();
wordDictionary.Add(word, 1);
//Loop through each key in the dictionary
foreach(String word in wordDictionary.Keys)
