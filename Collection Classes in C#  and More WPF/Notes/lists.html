IT 330

Lists

Background

Strongly typed collection of objects that can be accessed by index. Provides methods to search, sort, and manipulate lists.

Other than a few specific scenarios (bit shifting and stream manipulation), you really want to use Lists over Arrays.  Lists offer many helpful methods to work on objects in the collection, type safety and the ability to resize.

Initialize List

Notice how we initialize a list of type string:

	    List<string> cities = new List<string>();
            cities.Add("Chicago");
            cities.Add("San Diego");
            cities.Add("Atlanta");
            cities.Add("Boston");
            cities.Add("Seattle");
            cities.Add("Dallas");
            cities.Add("Kalamazoo");
The type is defined between the <>.  What if you wanted a list of Person objects?
	List<Person> cities = new List<Person>();

List has methods Sort() and Reverse() to reorder the collection. Ordering objects requires you to create a comparer object that implements ICompare.  We'll see some more advanced C# features called LINQ that makes it easy to query and manipulate collections of objects in a future talk.
