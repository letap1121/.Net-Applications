Arrays

Background

An array is a fixed sized collection of items of one type that is accessible through its index number.  Arrays are a core component of most programming languages.

Classic Arrays

Arrays are declared with type followed by [].  Arrays are fixed length so if you need to resize, you need to create a new one.

int[] items = new int[3];
//Set each item in the array
items[0] = 5;
items[1] = 2;
items[2] = 7;

Arrays are very fast and easy to work with as long as you don't get into a resize situation.

Examples: SimpleArrays

ArrayList

Not being able to resize an array can be problematic.  ArrayLists are a flexible alternative that allow you to add and remove items as you need them.  Main drawback to ArrayLists is that they only store the object type.  You have to explicitly cast all the objects stored in the ArrayList back to its core type in order to use them.

private ArrayList col = new ArrayList();

//Adding a new Person to the list
col.Add(new Person());

//Notice the explicit cast when obtaining the item
Person p = (Person)col[i];

One of the advantages of ArrayList is you can contain multiple types in the collection.  You need to take special care when accessing the items.

Summary

  • Arrays are strongly typed
  • ArrayList is not strongly typed.
  • Items in Arrays have to be of same data type (int, string, or any class).
  • Items in ArrayList can contain objects of any type. However, you have to manually cast each item.
  • Arrays are fixed size.
  • ArrayList can grow and shrink during runtime.

With C# 2.0, we now have better infrastructure for Collection Classes that overcome some of the limitations of ArrayList.
