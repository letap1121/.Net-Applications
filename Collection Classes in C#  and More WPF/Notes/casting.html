Casting and Conversion

Background

Casting and Conversion from one type to another are an important part of any programming language.  C#, being a type safe language, has rules that have to be followed when converting between types.  Lets explore these types and how they behave in these situations.

  • Numeric datatypes can be converted from one type to another by using a Java-style cast.
      double x = 4.38;
int n = (int) y;
  • Numeric casts can be either widening or narrowing.
     
  • A widening cast is guarenteed not to lose information.
     
  • A narrowing cast risks losing information. For example in
      short x = 10000;
byte y = (byte) x;
  • An enum can always be cast to its underlying int type or any type to which int widens.
     
  • A cast is required for a narrowing conversion; a cast is not needed for a widening conversion:
      int n = 169;
double x = n; 
long y = n;
  • We already know about int.Parse and int.TryParse methods. The Parse and TryParse methods also exist for these numeric types:
     
      long  short  byte  double  float  decimal

  • Some conversion functions of the System.Convert class:
     
      ToBase64CharArray  ToBase64String  ToBoolean  ToByte  ToChar  ToDate
       
      ToDateTime  ToDecimal  ToDouble  ToInt16  ToInt32  ToInt64  ToSByte
       
      ToSingle  ToString  ToUInt16  ToUInt32  ToUInt64

char[ ] char[ ], string

1 By definition, every data type widens to itself.

2 Conversions from int, uint, long, ulong, or decimal to float or double might result in loss of precision, but never in loss of magnitude. In this sense they do not incur information loss.

Widening conversions always succeed at run time and never incur data loss. They are always performed implicitly, even if no cast is used.

Narrowing Casts

These are the standard narrowing conversions:

  • The reverse directions of the widening conversions in the preceding table (except that every type widens to itself).
     
  • Conversions in either direction between Boolean and any numeric type.
     
  • Conversions from any numeric type to any enumerated type (enum).
     
  • Narrowing conversions do not always succeed at run time, and can fail or incur data loss. An error occurs if the destination data type cannot receive the value being converted. For example, a numeric conversion can result in an overflow.
